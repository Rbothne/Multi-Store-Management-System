Multi-Store Management System (SQL, HTML, CSS, REST, Elixir, Phoenix)                      
A web application to manage multiple store inventories and employee registers utilizing a separate PostgreSQL backend.
Allows for modification and query of store database entries in relation to retail price, wholesale price, stock, and vendors. Employee data modification is prevented through JSON return, accessible only through backend site.
NOTE: to setup server see Phoenix documentation. Requires postgreSQL server and relations and provided hosting on modifiable ports.

To see commands relative to Phoenix server startup, view interior readme files.
